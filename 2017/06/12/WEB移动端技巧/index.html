WEB移动端技巧

移动端布局

一般在写移动端项目我都采用的是Flex+rem布局，以这种模式布局的话就可以解决我们在移动端项目实现的各种问题。

flex布局

为什么要用Flex布局呢，在传统的PC端布局中，我们最喜欢用的是float+postion等等，有时候要自适应移动端的布局中，存在各种问题。所以推荐使用flex，在这我推荐阮一峰老师的《Flex 布局教程：语法篇》文章

rem

CSS3的出现，他同时引进了一些新的单位，包括我们今天所说的rem单位。在W3C官网上是这样描述rem的——“font size of the root element”！也就是说它根据html根元素的字体大小变而改变的，当然与之对应的是em单位，em单位是以所在的父元素的字体大小而变的;

设置移动端宽高度相等

在移动端项目中，往往一些详情页中，图片一般般往往显示在最上面而且宽高相等的正方形图片，要自适应手机屏幕的时候，大家常用的方法就是宽度 width:100%;（这个方法的前提就是图片宽高相等的情况下），但有没有想到如果图片宽高不相等，或者说一个设计稿的内容板块div宽高相等的情况下，难道用js么？这样很麻烦，实际上不用，废话那么多上css代码

至于width: 100%;大家都很明白 主要是height:0; padding-top:100%; 为什么要设置高度为0呢，很明显我要的是宽度和高度相等100%！如果设置高度为100%的时候那就不行了，毕竟高度100%不是根据宽度来计算的。设置高度为0主要的是为了让padding-top属性设置100%值的时候，让这个属性根据宽度计算，这样就完全宽高相等的样式操作了。
